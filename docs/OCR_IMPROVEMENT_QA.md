# OCR 개선 옵션 — 질문·답변 (확인 후 적용)

아래 질문에 답해 주시면, 답에 맞춰 코드/설정을 적용할 수 있도록 정리해 두었습니다.  
**답변 예시:** `Q1 예`, `Q2 0.6`, `Q3 아니오` 처럼 번호와 답만 적어 주셔도 됩니다.

---

## Q1. PDF 업로드할 때 최대 용량 제한을 둘까요?

- **질문:** "한 번에 올릴 수 있는 PDF 크기를 제한할까요? (너무 큰 파일은 서버/메모리 부담이 커집니다.)"
- **답변:** `예` / `아니오`  
  - **예** → 최대 허용 크기(MB)를 정해 주세요. (예: `50` 이면 50MB 제한 적용)
  - **아니오** → 제한 없이 현재처럼 둡니다.

**답변 시 적용:** `예` + 숫자(예: 50) → FastAPI에서 해당 MB 제한 적용.

---

## Q2. 인식 신뢰도 기준(drop_score)을 바꿀까요?

- **질문:** "OCR이 애매하게 읽은 글자는 버리고, 확실한 것만 쓰고 싶나요? (기본 0.5)"
- **답변:** `기본(0.5)` / `더 엄격(0.6 또는 0.7)` / `더 관대(0.3 또는 0.4)`  
  - **기본** → 0.5 유지.
  - **더 엄격** → 0.6 또는 0.7 등 원하는 숫자.
  - **더 관대** → 0.3 또는 0.4 등 원하는 숫자.

**답변 시 적용:** 선택한 숫자로 `get_ocr_engine(drop_score=...)` 에 반영.

---

## Q3. 의회/조례 등 전용 글자 사전 파일을 쓸까요?

- **질문:** "특정 용어·기호만 더 잘 인식하게 하고 싶다면, '한 줄에 한 글자' 텍스트 파일로 사전을 넣을 수 있습니다. 사용할까요?"
- **답변:** `아니오` / `예, 파일 경로: ...`  
  - **아니오** → 커스텀 사전 없이 현재처럼 둡니다.
  - **예** → 사전 파일 경로를 알려 주세요. (예: `backend/data/rec_custom_dict.txt`)  
    - 파일이 없으면 빈 파일 또는 예시 내용으로 만들어 둘 수 있습니다.

**답변 시 적용:** `get_ocr_engine(rec_char_dict_path="...")` 에 연결하고, 필요 시 예시 사전 파일 생성.

---

## Q4. 디지털 PDF(텍스트가 이미 있는 PDF)는 OCR 없이 바로 텍스트만 뽑을까요?

- **질문:** "글자가 이미 들어 있는 PDF는 이미지로 바꾸지 않고, 바로 텍스트만 추출할까요? (빠르고 품질 좋음)"
- **답변:** `아니오` / `예`  
  - **아니오** → 지금처럼 모든 PDF를 이미지로 바꾼 뒤 OCR만 사용.
  - **예** → 먼저 PDF에서 텍스트를 추출 시도하고, 없거나 부족하면 OCR 경로로 처리하는 로직 추가.

**답변 시 적용:** 디지털 PDF 텍스트 추출(예: PyMuPDF 등) 후, 없을 때만 기존 OCR 파이프라인 호출하도록 수정.

---

## Q5. GPU 메모리 부족(OOM)이 나면 rec 배치를 줄일까요?

- **질문:** "한 페이지에 글자가 많을 때 GPU 메모리 부족이 나면, 한 번에 인식하는 개수(rec_batch_num)를 30 → 10처럼 줄일까요?"
- **답변:** `아니오` / `예, 10` (또는 원하는 숫자)  
  - **아니오** → 기본값 30 유지.
  - **예** → 10 등 원하는 숫자.

**답변 시 적용:** `get_ocr_engine(rec_batch_num=...)` 인자 추가 후 해당 값으로 설정.

---

## Q6. 스캔 품질이 나쁜 문서를 위해 이미지 전처리(대비·밝기 등)를 넣을까요?

- **질문:** "흐리거나 어두운 스캔본 때문에 인식률이 떨어지나요? 그렇다면 이미지 보정(대비/밝기)을 OCR 전에 넣을 수 있습니다."
- **답변:** `아니오` / `예`  
  - **아니오** → 전처리 없이 현재처럼 둡니다.
  - **예** → PDF→이미지 변환 후, OCR 직전에 간단한 보정(예: CLAHE·정규화) 적용하는 단계 추가.

**답변 시 적용:** `pdf_service` 또는 OCR 호출 직전에 전처리 함수 추가 후 파이프라인에 연결.
